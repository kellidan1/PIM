The Problem

E-commerce teams suffer from product data chaos

😤

Scattered Data

Product details live in spreadsheets, emails, and directly in the e-commerce dashboard — no single source of truth.

🔄

Manual, Error-Prone Updates

Updating 500 products across WooCommerce by hand is slow, inconsistent, and breaks frequently.

📦

No Schema Enforcement

Different team members use different attribute names — "colour", "Color", "col" — for the same concept.

The Solution

One Master Record.
Infinite channels.

PIM Sync is a standalone product information hub. All product data is authored once, centrally validated, then pushed to WooCommerce (and future channels) on demand.

Architecture

Four-model data schema

📦

Product

title · slug · SKU · price · description · status (draft/pending/synced)

🏷️

Attribute

name · type (text / number / select) — globally defined schema

🔗

AttributeValue

productId · attributeId · value — unique junction with cascade delete

📋

SyncLog

Full audit trail of every WooCommerce push — status, message, timestamp

model Product {
  id       String @id
  sku      String @unique
  status   String // draft|pending|synced
  attributes AttributeValue[]
  syncLogs SyncLog[]
}

model AttributeValue {
  …
  @@unique([productId, attributeId])
}

Key Features

A complete product lifecycle

🖥️

Master Dashboard

Real-time stats: total products, synced count, pending queue, attribute count. Live sync activity feed.

📊

Product Catalog

Single or bulk sync to WooCommerce. SKU-based create-or-update. Automatic status reset on edit.

Every sync event logged with status, message, and timestamp. Full traceability for compliance.

⚙️

Settings Hub

WooCommerce API credential configuration. Environment-based secrets via .env file.

Sync Engine

How the synchronization works

01

User triggers sync (Single or Bulk)

POST /api/products/[id]/sync or POST /api/sync-all

02

PIM fetches full product record from SQLite

Including all attribute values via Prisma relations

03

Check WooCommerce for existing SKU

GET products?sku= → decides CREATE or UPDATE path

04

POST or PUT to WooCommerce REST API v3

Maps PIM schema → WooCommerce attributes array format

05

Update status to 'synced' · Create SyncLog entry

Failed syncs are also logged with the error message
